## Spacing Scale

Declared values (must be multiples of 4) — identical to Phase 1, no changes:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge internal padding, `border-l-4` accent width |
| sm | 8px | Compact spacing (checkbox-to-label gap `gap-3`≈12px is the mockup exception below; filter list item gaps) |
| md | 16px | Card internal padding components (`p-4` on book cards), modal field gaps (`space-y-4`) |
| lg | 24px | Card grid gap (`gap-6`), section padding, modal padding (`p-6`), `gutter` token |
| xl | 32px | Page section vertical rhythm (`py-8`/`py-12` desktop), filter sidebar gap (`gap-8`) |
| 2xl | 48px | Empty-state vertical centering padding (`py-12`/`min-h-[40vh]`) |
| 3xl | 64px | Not used heavily in Phase 2 |

Exceptions:
- `gutter` (24px) — reused from Phase 1, applies to the catalog grid gap (`gap-gutter` / `gap-6`) and the sidebar-to-grid layout gap.
- `margin-mobile` (20px) / `margin-desktop` (40px) — page-edge margins for `/katalog` and `/katalog/{id}` main containers, matching the mockup's `px-margin-mobile md:px-margin-desktop`.
- Filter checkbox row uses `gap-3` (12px, off-grid) — copied verbatim from the `katalog_buku_biblio` mockup's `<label class="flex items-center gap-3 ...">`. Treat as an inherited mockup exception, do not "fix" to `gap-2`/`gap-4`.
- **Cross-reference:** `gap-3` (12px), `margin-mobile` (20px), and `margin-desktop` (40px) were already reviewed and approved as off-grid exceptions in Phase 1's `01-UI-SPEC.md` sign-off (see its Spacing Scale exceptions, lines 130-133). Phase 2 inherits these as pre-approved — no re-litigation needed in this or future phases.
- 44px minimum touch target — applies to: the "Kelola"/"Jelajah" tab toggle buttons on mobile, the modal close (`close`) icon button, and any icon-only filter/clear buttons. `min-w-[44px] min-h-[44px]` or equivalent padding.
- Book card cover aspect ratio `aspect-[2/3]` is a fixed visual ratio, not a spacing token — keep as-is from the mockup (DESIGN.md "Cards" section: "A clear image ratio for book covers (2:3)").

## Typography

Phase 2 reuses **exactly the same 4-size type scale Phase 1 declared** — no new sizes or weights introduced, and no 5th/6th/7th size added. Reference table (from `tailwind.config.js`, already present):

| Role | Size | Weight | Line Height | Font | Usage in Phase 2 |
|------|------|--------|-------------|------|------|
| headline-md | 24px | 600 (semibold) | 32px (1.33) | Playfair Display | Page title "Katalog Buku" (`/katalog`); book card title (`<h3>` in `BookCard`); `/katalog/{id}` book title (detail view); modal dialog title ("Tambah Buku" / "Edit Buku") |
| body-md | 16px | 400 (regular) | 24px (1.5) | Inter | Body copy: detail view metadata (penulis, full ISBN row, tahun terbit, kategori), modal field values, empty/error state body text, book card author line, "Menampilkan {N} buku" result count |
| label-md | 14px | 600 (semibold), 0.05em letter-spacing, uppercase where used for section headers | 16px (1.14) | Inter | "Pinjam Buku" button, "Tambah Buku"/"Tambah Salinan"/"Simpan Buku"/"Simpan Perubahan"/"Batal" buttons, input labels in modal/forms, filter sidebar section headers ("Kategori"/"Ketersediaan", uppercase), section headers on detail view and Kelola tab ("Salinan Buku", "Tambah Salinan", "Kelola Katalog" — same uppercase treatment as Phase 1's nav links) |
| label-sm | 12px | 400 (regular) | 16px (1.33) | Inter | Availability badge text ("Tersedia"/"Tidak Tersedia"), category tag micro-label, ISBN code (`font-mono` per mockup), table headers AND table cell text (Kelola/Salinan tables) |

**Weight contract:** exactly **2 weights** — same as Phase 1, no 3rd weight introduced. **400 (regular)** for body copy, badges, helper text, and table content; **600 (semibold)** for page/card/dialog/detail titles, buttons, input labels, and section headers (`label-md`). `label-sm` is rendered at 400 (see Design System note above re: Inter 500 not being loaded).

**Heading line-height:** 1.33 (headline-md = 32/24) — matches Phase 1's accepted range (1.2–1.33).
**Body line-height:** 1.5 (body-md = 24/16) and 1.33 (label-sm = 16/12) — both within Phase 1's accepted range; `label-md` uses 1.14 (16/14) as in Phase 1, appropriate for short uppercase labels/buttons.

**Re-map note (per UI checker revision):** Phase 2's original draft introduced `headline-lg`/`headline-lg-mobile` (32/28px), `headline-sm` (20px), and a separate `body-sm` (14px) role — exceeding the 4-size maximum. These have been consolidated onto the 4 sizes above: the page title now uses `headline-md` (not `headline-lg`), book card titles and detail-view section headers use `headline-md`/`label-md` respectively (not `headline-sm`), and all former `body-sm` usages (author line, result count, table cells) are re-mapped to `body-md` or `label-sm` per the Layout & Navigation Contract below. `headline-lg`/`headline-lg-mobile`/`headline-sm` remain defined in `tailwind.config.js` from Phase 1 but are **not** part of Phase 2's active type scale (Phase 1's `headline-lg` continues to apply only to the existing "Biblio" logo/wordmark, unchanged).

| Secondary (30%) | `#FFFFFF` (`surface-container-lowest`, cards/filter panels/modal) bordered with `#E8E4D9` (`paper-shadow`); `#F5F3F5`/`#EFEDEF` (`surface-container-low`/`surface-container`, table stripe rows, disabled "Tidak Tersedia" button surface) | Book cards, filter sidebar panels, modal dialog surface, Kelola admin table row/header backgrounds, Salinan copy table |

| Destructive | `#BA1A1A` (`error`) | "Hapus" (delete) button in Kelola table actions, delete confirmation dialog text/border, form validation error messages (ISBN format, tahun_terbit range, required fields) in add/edit `buku` modal and Tambah Salinan form |

**Binary availability badge colors (D-05):**
- "Tersedia": `bg-surface-container-lowest/90` chip with `text-sage-green` label text + `border border-paper-shadow` (catalog card overlay, per mockup) OR `bg-sage-green/10 text-sage-green` inline chip (detail view / table cell) — sage green is the "available" signal throughout.
- "Tidak Tersedia": `bg-surface-container-highest/90` chip with `text-on-surface-variant` label text + `border border-outline-variant` (catalog card overlay) OR `bg-surface-container text-on-surface-variant` inline chip (detail view / table cell) — neutral grey, NOT alert-crimson (D-05 says "grey/neutral", not destructive-red — `alert-crimson` stays reserved for Phase 3/4 overdue states per Phase 1's note).

**Category-color mapping for placeholders & left-edge accents (D-13/D-14, Claude's Discretion — resolved):**

Since `kategori` is free-text VARCHAR(100) (D-09) with no fixed enum, define a **3-color rotation** covering the seeded categories (Fiksi, Non-Fiksi, Referensi per D-11) plus a deterministic fallback for any future category:

| Kategori (seed) | Left-edge accent (`border-l-*`) | Placeholder cover background | Placeholder icon color |
|---|---|---|---|
| Fiksi | `border-l-ink-blue` (`#112240`) | `bg-ink-blue/10` | `text-ink-blue` |
| Non-Fiksi | `border-l-antique-gold` (`#C5A059`) | `bg-antique-gold/10` | `text-antique-gold` |
| Referensi | `border-l-secondary-fixed` (`#e4e2de`, neutral grey) | `bg-surface-container-high` | `text-on-surface-variant` |
| Any other / unmapped `kategori` (fallback) | `border-l-sage-green` (`#8A9A5B`) | `bg-sage-green/10` | `text-sage-green` |

When a book is "Tidak Tersedia" (D-14), the left-edge accent is **overridden to neutral** regardless of category: `border-l-secondary-fixed` (matches mockup's "Book Card 3 (Referensi - Dipinjam)" treatment, which uses `border-l-secondary-fixed` for an unavailable Referensi book — i.e. unavailability visually "mutes" the category accent to the same grey used for Referensi).

**Placeholder cover component (`BookCoverPlaceholder`, D-13):** a `div` with `aspect-[2/3] w-full flex items-center justify-center` + the category background color from the table above, containing a centered Material Symbols `menu_book` icon (`text-5xl` on catalog cards, `text-7xl` on the detail view per D-15) in the matching icon color. No `<img>` tag, no external image URLs (unlike the mockup which uses placeholder `<img>` covers).

Accent reserved for: category tag labels, filter checkboxes, search input focus, "Tambah Buku"/"Tambah Salinan" buttons, active Jelajah/Kelola tab indicator. Do NOT use Antique Gold for the "Pinjam Buku" button (that's Sage Green) or for destructive/delete actions (that's `error`).

**Destructive actions in this phase:**
- **Hapus Buku** (delete master `buku` record, CAT-03): confirmation dialog required (see rows above). ## Layout & Navigation Contract

### Route structure (D-01, D-02, D-03)
- `/katalog` — shared route, single page component. Renders a tab/toggle row (`Jelajah` | `Kelola`) **only for `pustakawan`**; mahasiswa always sees the "Jelajah" grid with no toggle visible (tab state defaults to/locks at "Jelajah" for mahasiswa — do not render the toggle UI at all for `peran === 'mahasiswa'`, per D-01 "no new nav items" and role-gating pattern from `AppShell.jsx`).
- `/katalog/:id` — shared detail route (D-02), same component tree for both roles; pustakawan additionally sees the "Tambah Salinan" form (D-04, D-07).
- Update `frontend/src/router.jsx`: replace `{ path: 'katalog', element: <ComingSoonPage title="Katalog Buku" /> }` with `{ path: 'katalog', element: <KatalogPage /> }` and add `{ path: 'katalog/:id', element: <BukuDetailPage /> }`, both inside the existing `AppShell`-wrapped `ProtectedRoute` group.
- No nav changes to `AppShell.jsx` — the existing "Katalog" link (`/katalog`, `search` icon) for both `MAHASISWA_NAV` and `PUSTAKAWAN_NAV` already points here.

### `/katalog` — "Jelajah" (browse grid, mahasiswa default + pustakawan tab)

**Page header:**
- `text-headline-md font-headline-md text-primary` — "Katalog Buku" as the page-level title (new, sits above the tab toggle for pustakawan; for mahasiswa this is the only heading, directly replacing the mockup's "Katalog Terkini").

Each tab button min 44px height for touch target. `role="tablist"` / `role="tab"` / `aria-selected` for accessibility.

**Layout (matches mockup structure):**

This filter is **client-side only** (applied to the already-fetched page of results) since it depends on the derived rollup, not a raw DB column — document this distinction in the plan so the executor doesn't attempt to pass an `availability` query param to the backend.

### Accessibility
- All icon-only buttons (`close`, `edit`, `delete`, hamburger if applicable) have `aria-label` per the Copywriting Contract.
- Category filter checkboxes and availability radio buttons are real `<input type="checkbox">`/`<input type="radio">` with associated `<label>` (click target = full row, per mockup's `<label class="flex items-center gap-3 cursor-pointer">`).
- Tab toggle uses `role="tablist"`/`role="tab"`/`aria-selected="true|false"`.
- Modal uses `role="dialog"` `aria-modal="true"` `aria-labelledby` pointing to the modal title; focus moves to the first input on open and returns to the triggering button on close; `Escape` closes.
- Toast/snackbar ("coming soon" message) uses `role="status"` `aria-live="polite"`.
- Disabled "Tidak Tersedia" buttons retain `aria-disabled="true"` and are not focusable via tab order beyond standard `disabled` semantics.
