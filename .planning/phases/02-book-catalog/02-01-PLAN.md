<objective>
Stand up the read side of the catalog data layer: Pydantic schemas, a `/api/buku` router with search/filter/category/detail endpoints, seed data, and tests. This is the thinnest real end-to-end data path for the mahasiswa catalog-discovery slice (CAT-01, CAT-02) — the frontend in Plan 02 consumes these exact endpoints.

Purpose: Without a real, seeded, search/filter-capable read API, the mahasiswa cannot discover books — the core capability this phase delivers.
Output: `backend/app/schemas/buku.py`, `backend/app/routers/buku.py` (read endpoints only), registration in `main.py`, extended `seed.py`, and `backend/tests/test_buku.py` (GREEN).
</objective>

## Phase Goal

**As a** mahasiswa, **I want to** search and filter the book catalog by title, author, ISBN, and category, **so that** I can find a book I want to borrow.
