## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Elevation of Privilege | Kelola tab / modal / Tambah Salinan visibility | mitigate | UI gated on `peran === 'pustakawan'`, but security relies on Plan 02-03's server-side 403 — a tampered client still cannot mutate. Frontend gate is UX-only and documented as non-authoritative |
| T-02-12 | Tampering (XSS) | rendering judul/penulis/kategori in table, modal, detail | mitigate | React text children auto-escape; no dangerouslySetInnerHTML; datalist options are app-controlled category strings |
| T-02-13 | Repudiation / data loss | delete flow | mitigate | Destructive delete requires explicit confirmation dialog; FK-safe 409 surfaced as an informational variant with no delete button (cannot accidentally orphan copies) |
| T-02-14 | Information Disclosure | err.response.data.detail surfaced to user | accept | Detail strings are intentional Indonesian user-facing messages from the API (no stack traces leaked by FastAPI HTTPException) |
| T-02-SC | Tampering | npm installs | accept | No new npm packages — reuses axios/react-router/material-symbols from Phase 1; no install task present |
</threat_model>
