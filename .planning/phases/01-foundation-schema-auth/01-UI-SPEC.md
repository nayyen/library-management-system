## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, tight inline padding |
| sm | 8px | Compact element spacing (e.g. label-to-input gap, `space-y-2`) |
| md | 16px | Default element spacing, input vertical padding (`py-2`/`py-3`), card internal gaps |
| lg | 24px | Section padding, card padding (`p-6`), form `space-y-6` |
| xl | 32px | Layout gaps, logo block bottom margin (`mb-8`) |
| 2xl | 48px | Major section breaks (not used heavily in Phase 1's narrow scope) |
| 3xl | 64px | Page-level vertical rhythm (not used heavily in Phase 1's narrow scope) |

Exceptions:
- `margin-mobile` (20px) and `margin-desktop` (40px) — canonical page-edge margins from DESIGN.md, applied as `px-margin-mobile md:px-margin-desktop` on the body/header/main containers. Off the 8pt grid by design; do not "fix" to 16/24/32.
- `gutter` (24px) — canonical grid gutter token from DESIGN.md, equivalent to `lg` above but kept as a named token for grid/flex `gap-gutter` usage to match the mockups' class names.
- 44px minimum touch target — any icon-only interactive element on mobile (e.g. hamburger/menu toggle for the authenticated shell's nav, account icon button) must have a minimum tappable area of 44x44px (`min-w-[44px] min-h-[44px]` or equivalent padding), even though 44 is not a multiple of 8. This applies specifically to icon-only buttons in the header at <768px.

## Typography

4 sizes declared for Phase 1's scope (login, register, authenticated shell, welcome card, placeholder pages):

| Role | Size | Weight | Line Height | Font | Usage |
|------|------|--------|-------------|------|-------|
| headline-md | 24px | 600 (semibold) | 32px (1.33) | Playfair Display | Card titles ("Masuk ke Akun Anda", "Buat Akun Baru"), welcome heading ("Selamat datang, {nama}"), "Segera Hadir" placeholder heading |
| body-md | 16px | 400 (regular) | 24px (1.5) | Inter | Body copy, input text, placeholder body copy, error/helper messages |
| label-md | 14px | 600 (semibold), 0.05em letter-spacing, uppercase where used for nav | 16px (1.14) | Inter | Input labels ("Email Institusi", "Kata Sandi", "Nama"), primary buttons ("Masuk", "Daftar"), nav links (Katalog, Pinjaman, Dashboard, Anggota) |
| label-sm | 12px | 400 (regular) | 16px (1.33) | Inter | Secondary/helper text, role badge text (mahasiswa/pustakawan per D-06), "Sudah punya akun?" / "Belum memiliki akun?" link rows |

**Weight contract:** This phase declares exactly **2 font weights**: **400 (regular)** and **600 (semibold)**. 400 is used for body copy, input text, and secondary/helper text (`label-sm`); 600 is used for headings, primary buttons, input labels, and nav links (`label-md`, `headline-md`). Do not introduce any other weight.

**Heading line-height:** 1.2–1.33 range (headline-md = 32px/24px = 1.33) — acceptable per DESIGN.md's headline tokens; do not compress further.
**Body line-height:** 1.5 (body-md = 24px/16px) — matches the recommended default.

The Playfair Display `headline-lg` (32px/700, used for the "Biblio" wordmark on the login screen and the nav brand) is part of the existing mockup and may be reused as-is for the logo/wordmark — it is not a "5th size" for new UI, just the existing brand lockup.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FBF9FB` (`surface` / `background`, warm cream) | Page background (`<body>`), main content area background |
| Secondary (30%) | `#FFFFFF` (`surface-container-lowest`, cards) and `#F5F3F5` (`surface-container-low`, side panels/nav backgrounds), bordered with `#E8E4D9` (`paper-shadow`) — plus Navy `#0A192F`/`#000000` (`primary`/`primary-container`) for headings, primary button surfaces, and the nav brand wordmark/icon | Login/register card surfaces, authenticated shell header/sidebar backgrounds, card borders, primary button fill, heading text color, nav brand |
| Accent (10%) | `#C5A059` (Antique Gold, `antique-gold`) | **Reserved strictly for:** (1) input focus ring/border (`focus:border-antique-gold`), (2) "Daftar sekarang" link (login → register) and "Sudah punya akun?" link (register → login), (3) active nav link indicator (bottom border/underline on the current section in the authenticated shell nav), (4) the login/register card's left-edge "spine decoration" accent strip (`bg-antique-gold/20`) |
| Destructive | `#BA1A1A` (`error`) | Form validation error text/borders only (e.g. "Email atau kata sandi salah.", "Kata sandi minimal 8 karakter.", "Email sudah terdaftar.") — applies to inline error messages and the input's error-state border |

Accent reserved for: input focus border/glow, "Daftar sekarang" / "Sudah punya akun?" cross-links, active nav item underline, login/register card spine decoration. Do NOT use Antique Gold for primary buttons, body text, or general interactive hover states beyond what's listed.

**Noted for future-phase consistency (not used in Phase 1):** `#93272C` (Alert Crimson, `alert-crimson`) is reserved for overdue/blocked-account states starting in Phase 3/4 — do not repurpose it for Phase 1 errors (use `error` `#BA1A1A` instead).

**Role badge colors** (welcome card, D-06): use existing token pairs to differentiate mahasiswa vs pustakawan without introducing new colors —
- `mahasiswa` badge: `bg-secondary-container` (`#E1DFDC`) text `on-secondary-container` (`#636360`)
- `pustakawan` badge: `bg-primary-container` (`#0D1C32`) text `on-primary-container` (`#76849F`) — echoes the "Librarian Access" treatment in the dashboard mockup's sidebar avatar block

## Layout & Navigation Contract

**Login (`/login`) and Register (`/register`):**
- Reuse the login mockup's centered single-column layout: logo block (Material Symbols `menu_book` icon, "Biblio" headline-lg wordmark, "Sistem Manajemen Perpustakaan" tagline) + card (`bg-surface-container-lowest`, `border border-paper-shadow`, `rounded-xl`, `p-6 md:p-8`, soft shadow `shadow-[0_20px_40px_-15px_rgba(0,0,0,0.05)]`, left-edge spine decoration `absolute left-0 top-0 bottom-0 w-1 bg-antique-gold/20`).
- Register card: same structure, with "Nama" field inserted above "Email Institusi", card title "Buat Akun Baru", submit button "Daftar", cross-link "Sudah punya akun? Masuk".
- Both pages must render correctly at 375px width (NF03) — the mockup's `max-w-md` centered container with `p-margin-mobile md:p-margin-desktop` already satisfies this; verify input fields and buttons remain full-width and tappable at 375px.

**Authenticated shell (header/nav, D-05):**
- Top header bar (`bg-background`, `border-b border-outline-variant`, height 64px / `h-16`): left side = Biblio brand (icon `local_library` + "Biblio" wordmark, `headline-md`/`headline-lg` per existing mockup styling, `text-primary`); nav links "Katalog", "Pinjaman", "Dashboard", "Anggota" (`label-md`, `text-on-surface-variant`, hover → `text-antique-gold`); right side = account icon button (`account_circle`, Material Symbols, filled).
- Accessible labels for icon-only buttons (required, `aria-label`): the `account_circle` account button uses `aria-label="Akun"`; the `menu` hamburger menu toggle (mobile nav) uses `aria-label="Buka menu navigasi"`.
- Active nav link gets `border-b-2 border-antique-gold` underline + `text-primary` (per dashboard mockup's "Dashboard" link styling) — this is the one "active state" indicator for Phase 1's shell.
- **Role-based nav visibility:** show all 4 links (Katalog, Pinjaman, Dashboard, Anggota) to `pustakawan`. For `mahasiswa`, "Dashboard" and "Anggota" are pustakawan-only per the PRD's role model — Phase 1 should still render the shell for both roles, but nav link visibility should be role-aware from the start (mahasiswa sees Katalog + Pinjaman; pustakawan sees all four) since the shell is the reusable layout for Phases 2-5. If this role-based filtering is ambiguous at execution time, default to showing all 4 links for both roles in Phase 1 and let Phase 5 (Dashboard/Members) tighten visibility — but prefer filtering now since it's a one-line conditional.
- On mobile (<768px), nav links collapse — at minimum the header brand and the account icon button must remain visible and tappable (44x44px min touch target per Spacing Scale exceptions). A hamburger/menu icon-button (`menu`, Material Symbols) opening the nav links in a drawer/sheet is acceptable but not mandated by any Phase 1 success criterion — a simple stacked/visible nav under the header on mobile is sufficient if simpler to implement within the timeline.
- Sidebar (desktop dashboard layout from `dashboard_pustakawan_biblio/code.html`) is **out of scope for Phase 1** — that's a Phase 5 (Dashboard) concern. Phase 1's shell is header-only.

**Landing/welcome route (`/` after login, D-06):**
- Centered card (reuse the login/register card's `bg-surface-container-lowest border border-paper-shadow rounded-xl p-6 md:p-8` styling) containing: `headline-md` "Selamat datang, {nama}", a role badge (see Color section above for badge token pairs — text content "Mahasiswa" or "Pustakawan", `label-sm`, `rounded` per DESIGN.md's "Status Tags: 2px or 4px radius"), and a "Keluar" logout button (Ghost style per DESIGN.md: `border border-primary text-primary`, transparent background, `rounded` 4px).
- Same layout for both roles — only the badge text/color token pair differs.

**"Coming soon" placeholder pages (D-05):**
- Rendered inside the authenticated shell (header visible, content area shows the placeholder) for any nav route not yet built (Katalog, Pinjaman, Dashboard, Anggota all point here in Phase 1).
- Centered content: `headline-md` "Segera Hadir" + `body-md` "Halaman ini sedang dalam pengembangan. Fitur akan tersedia di fase berikutnya." Optional: a relevant Material Symbols icon (e.g. `construction` or `auto_stories`) above the heading at a large size (`text-6xl`, `text-outline-variant` or `text-secondary`) for visual interest, echoing the login logo treatment.

**Buttons (per DESIGN.md component spec):**
- Primary: solid `bg-primary text-on-primary`, `rounded` (4px), `py-3`, `label-md` — used for "Masuk", "Daftar".
- Ghost: `border border-primary text-primary`, transparent background, `rounded` (4px) — used for "Keluar" (logout).
- All buttons and inputs use 4px radius (`rounded` = `0.25rem` per the `borderRadius.DEFAULT` token — note the mockup's inline Tailwind config used `0.125rem` for DEFAULT; **use the DESIGN.md frontmatter values** as canonical: `DEFAULT: 0.25rem` (4px), `lg: 0.5rem` (8px) for cards).

**Inputs (per DESIGN.md component spec):**
- Bottom-border-only style: `border-0 border-b border-paper-shadow`, `bg-transparent`, `pl-10` (icon space), `py-2`.
- Label always visible above input, `label-md` (per mockup) — note DESIGN.md's prose says `label-sm` for input labels but the actual login mockup uses `label-md` (`font-label-md text-label-md`) for "Email Institusi" / "Kata Sandi" labels. **Use `label-md` for input labels** (mockup is canonical over prose description for this detail).
- Focus state: `focus:border-antique-gold`, `focus:ring-0` (no default browser ring — the gold border IS the focus indicator).
- Error state (login failure, duplicate email, password too short): border changes to `border-error` (`#BA1A1A`), with an inline `body-sm`/`label-sm` error message below the field in `text-error`.
- Leading icon inside input (Material Symbols Outlined, `text-outline-variant`, e.g. `mail`, `lock`, `person`).
